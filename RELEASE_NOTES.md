# 3D 關原之戰 — 電視級歷史節目　v1.0.0

> 慶長五年九月十五日（1600）· 東軍 對 西軍

以 [Three.js](https://threejs.org/) 打造的單頁互動式 3D 歷史戰役節目，靈感來自「3D 香港保衛戰」。純前端、免 build、可自由運鏡，依時間軸（辰刻 08:00 ~ 未刻 14:00）演示關原之戰雙方陣型、行軍、家紋軍旗與戰場特效，像一集電視特別節目。

線上觀看：**https://leorepro.github.io/BattleOfSekigahara/**

---

## ✨ 主要功能

- **真實衛星地形**：採用關原一帶 SRTM 30m 真實 DEM 高程資料（41×41）建構地形網格，依真實海拔著色，並以雙線性內插貼地。涵蓋笹尾山、松尾山、南宮山、桃配山、天滿山等關鍵地形。
- **家紋幟旗**：12 名武將各自的家紋以 Canvas 程序化繪製為飄動軍旗，東軍（德川）藍、西軍（石田）紅，一眼分辨敵我。
- **逐鏡運鏡（節目模式）**：內建 9 個 storyboard 鏡位，自動運鏡搭配字幕旁白，重大事件（如小早川秀秋倒戈）字幕高亮，呈現電視節目般的敘事節奏。
- **事件卡 / 字幕旁白**：依時間軸推進顯示日期、事件標題（中／英）、旁白與相關指令標籤。
- **行軍路線**：部隊移動以方向箭頭與路線呈現，並做防重疊處理（decollide）。
- **天氣特效**：朝霧 → 放晴的動態 fog／天色／光照／曝光變化，貼地飄移霧層；鐵炮齊射的砲口閃光（加色粒子）與硝煙（ShaderMaterial 粒子系統）於交戰部隊位置噴發。
- **兵力儀表**：即時顯示東西兩軍兵力，部隊潰滅時淡出。

## 🎮 操作方式

- **節目模式**（預設）：自動運鏡 + 字幕，像看電視節目。按「🕹 自由運鏡」切換為手動。
- **自由運鏡**：拖曳 = 旋轉鏡頭 · 滾輪 = 縮放 · 右鍵 = 平移。
- **時間軸**：拖曳底部 scrubber 跳到任一時刻（08:00 ~ 14:00）；可播放／暫停，並切換 0.5× ~ 4× 倍速。
- **點選部隊**：顯示武將、所屬陣營、即時兵力與家紋卡片。
- **配樂**：按「🔈 配樂」鈕播放背景音樂（音檔需自行放入 `assets/audio/bgm.mp3`，缺檔時不會出錯）。

## 🛰 資料來源

- **地形高程**：SRTM 30m DEM（透過 `fetch_dem.py` 由 opentopodata 取得，存於 `data/heightmap.js`）。
- **衛星底圖**：EOX Sentinel-2 cloudless。
- **戰役內容**：武將、家紋、地名、兵力與時序為本專案 `data/*` 自行整理。

## 🧱 技術架構

沿用「引擎／資料分離」設計：`src/engine/*` 為通用引擎，`data/*` 為關原戰役內容。使用全域 `THREE`（r128 UMD）+ 全域 `SEKI` 命名空間，無任何打包工具，純 HTML/JS 以相對路徑載入。

## 🚀 部署說明（GitHub Pages）

本專案透過 GitHub Actions（`.github/workflows/deploy.yml`）在每次 push 到 `main` 時自動部署為 GitHub Pages 靜態網站。

### 使用者需要手動做的事（僅需一次）

1. 進入 GitHub repo 的 **Settings**。
2. 左側選單點 **Pages**。
3. 在 **Build and deployment → Source** 下拉選單，選擇 **「GitHub Actions」**（不是「Deploy from a branch」）。
4. 完成。之後每次 push 到 `main`，workflow 會自動建置並部署；也可在 **Actions** 分頁手動以 **Run workflow** 觸發。

部署完成後網址為：

> **https://leorepro.github.io/BattleOfSekigahara/**

### 子路徑相容性

本網站全程使用**相對路徑**載入 `lib/`、`src/`、`data/`、`assets/` 等資源（見 `index.html`），不含任何以 `/` 開頭的絕對路徑，因此可在 GitHub Pages 的子路徑 `/BattleOfSekigahara/` 下正常運作，無需額外設定 base path。
