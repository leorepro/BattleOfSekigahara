# 溫泉關之戰（西元前 480 年）— 設計稿

**日期**：2026-06-15
**分支**：`feature/thermopylae-480bc`（待建立）
**狀態**：方向已與使用者確認，待 spec 審閱 → 進實作計畫

## 目標

在既有 3D 歷史節目專案（桶狹間 1560、關原 1600、諾曼第 1944）新增第四場戰役：**溫泉關之戰**。這是專案首場**古希臘冷兵器戰爭**，重用關原/桶狹間的冷兵器渲染路徑（**不走 `config.modern`**），但引入三個前所未有的核心挑戰：

1. **每名士兵都是完整個體**——希臘方約 7,000 人全部做成精細的重裝步兵（hoplite）個體，有圓盾、長矛、科林斯盔、緋紅披風、脛甲；不再是「一片色旗」的方陣方塊。
2. **極窄峽谷地形 + 古代海岸線還原**——西元前 480 年的溫泉關遠比今日狹窄（斯佩爾刻俄斯河三角洲淤積使海岸線北移數公里），須重建古海岸線（比照桶狹間「年魚市潟」古海岸做法）。
3. **真實人數 + 數字爭議並陳**——把「波斯號稱百萬 vs 現代史學估計」的心理戰本身做成敘事賣點。

維持與前三場一致的電視級節目感（自動運鏡、時間軸、字幕、兵力儀表、交戰拉鋸條、史料面板）。

## 已確認決策

- **敘事主線**：**C 戰略全局視角**——溫泉關不只是孤立的悲壯死戰，而是希波戰爭大棋局的關鍵一步：7,000 人（最後約 1,400 人）在峽谷死守三天，替雅典海軍爭取到薩拉米斯決戰的時間，結尾帶到希臘最終勝利（普拉提亞）。
- **史觀**：**真實人數、破除「300」迷思**。逐邦還原 298 斯巴達 + 約 7,000 聯軍；「300 壯士」標為後世/電影浪漫化。
- **士兵渲染**：希臘方**全部精細 hoplite 個體**；波斯方**真實人數 + 分層 LOD**（前線精細 + 中景實例 + 遠景人海剪影）。
- **波斯人數**：**C 並陳對比**——畫面以現代可信估計渲染，UI 同時顯示「希羅多德宣稱 vs 現代估計」兩組數字。
- **配色**：**方案 A 色系分群**——每個城邦/民族各有專屬色，但同陣營共用一個「色系家族」以保持敵我一眼可辨。**希臘聯軍 = 暖色系（紅/橙/金/赭）**、**波斯帝國 = 冷色系（藍/紫/青）**。仍與 east/west 地理及「勝方藍」慣例相容（波斯=east冷藍系、希臘=west暖紅系）。詳見 §3.1。
- **地圖**：全新下載真實 DEM（不可省略）+ 古海岸線還原。
- **切換列**：四頁皆顯示，依**年代排序**：溫泉關 -480 → 桶狹間 1560 → 關原 1600 → 諾曼第 1944。

## 架構原則（沿用既有，零回歸）

`src/engine/*` 與 `lib/*` 完全共用、不分叉。戰役差異透過**向後相容的全域 `SEKI.config`** 注入。前三場不設新欄位 → 全部 fallback 原行為。本場新增的旗標（`formationStyle:'phalanx'`、`ancientCoast` 等）前三場未設定 → **完全不受影響**。**不啟用 `config.modern`**（沿用戰國冷兵器渲染基礎再擴充）。

## 2. 時間軸與運鏡

- t = 距會戰首日的小時數，跨**三日**（約西元前 480 年 8/11–8/13）。`fmtTime` 改希臘曆敘述：「會戰第一日 · 正午」之類（無精確鐘點史料，用日/晨/午/夜分段）。
- **非線性時間軸**（沿用 Normandy `config.timelineAnchors`）：壓縮夜間、拉寬三場關鍵接戰，**第三日最後一役給最寬**。
- 相位 `phase(t)` 關鍵節點：
  1. **布陣**：希臘聯軍據中門、修復福基斯牆；波斯大軍抵達、薛西斯派使勸降（「交出武器」→「來拿吧 Molon labe」）。
  2. **第一日**：波斯米底軍正面強攻 → 斯巴達方陣盾牆+長矛輾壓，假退誘敵反殺。
  3. **第二日**：波斯「不死軍」（Immortals）投入仍不得寸進，希臘各邦輪番上陣。
  4. **第三日·背叛**：埃菲亞特斯獻安諾派亞山徑 → 不死軍夜間繞道 → **佛西斯守軍清晨被擊潰** → 希臘遭迂迴包圍。
  5. **第三日·殿後**：列奧尼達遣散聯軍，留 298 斯巴達 + 700 賽斯比 + 400 底比斯死守 → 長矛折斷改短劍/肉搏 → 全軍覆沒、列奧尼達戰死梟首。
  6. **尾聲·戰略全局**：鏡頭拉到全希臘地圖，帶出薩拉米斯海戰勝利 → 普拉提亞終結波斯入侵；溫泉關獅碑與西莫尼德斯銘文（「過路人，請告訴拉刻代蒙的人們……」）。
- storyboard 運鏡：峽谷全景（顯示窄道地形）→ 斯巴達方陣盾牆特寫 → 波斯人海壓上 → 山徑迂迴俯瞰 → 殿後死戰環繞 → 獅碑/全希臘地圖收尾。

**布陣前（背景）**
1. 薛西斯一世為雪馬拉松之恥，集波斯大軍架浮橋渡赫勒斯滂海峽西征

5. 適逢斯巴達卡尼亞祭與奧林匹克休戰，斯巴達僅派列奧尼達率前衛先行

10. 薛西斯遣騎兵偵察，見斯巴達人在牆前**梳髮、操練**，大惑不解

## 4. 士兵渲染系統（本作技術核心）

### 4.1 hoplite 個體構造（新模組 `src/engine/hoplite.js`）

`buildHopliteGeo(variant)` 產生合併 box 幾何，variant ∈ {spartan, ally, persian}。每名士兵**都具備**：

| 部件 | 希臘 hoplite | 波斯兵（對比版） |
|---|---|---|
| 盾 | 青銅圓盾 Aspis，正面刻 **Λ**（斯巴達）/ 各邦徽 | 柳條方盾 gerron |
| 主武器 | 長矛 Dory（2.5m，平端前刺，尾鐏） | 短矛 / 弓 |
| 頭部 | 科林斯全罩盔 + 鼻樑護板 + **馬鬃盔冠 λόφος** | 軟布帽 tiara |
| 軀幹 | 亞麻甲/青銅肌肉甲 | 鱗甲 + 花紋長袍 |
| 下肢 | 青銅脛甲 + 裸露小腿 | 長褲（anaxyrides）|
| 披風 | **緋紅斗篷 φοινικίς（斯巴達識別）** | 無/素色 |

材質：青銅金屬感（希臘）vs 布料花色（波斯），低面數（每兵約 150~250 面）。

### 4.2 個體差異系統（不再是複製人）

- InstancedMesh 共用幾何，用 **per-instance 偽隨機**（`sin(index)` 雜湊，因引擎禁用 `Math.random()`）微調：spear 角度、shield 高度、步距、披風/盔冠明度（±8%）。
- **前排英雄層**：最前一兩排（含列奧尼達、各邦指揮）改用 `models.js` 獨立 mesh（`hero-hoplite` kind），可單獨擺姿、陣亡倒地。

### 4.3 方陣與盾牆（Phalanx / Othismos）

- `formation.js` 新增 `config.formationStyle:'phalanx'` 分支（戰國/現代不受影響，向後相容）。
- 士兵排 **8 列縱深**密集方陣，**盾牆重疊**（每盾護己左半 + 左鄰右半）；後排長矛**架前排肩上**層層突出 → 經典「刺蝟」剪影。
- 方陣寬度受**地形隘道**卡死（正面僅容約 200 人並肩）→ 這正是史實「以寡敵眾」的物理基礎。

### 4.4 效能策略（分層 LOD）

| 層級 | 對象 | 渲染 | 量級 |
|---|---|---|---|
| 英雄層 | 列奧尼達、各邦指揮、前排數十人 | 獨立 mesh，可動畫/倒地 | ~50 |
| 精細層 | 希臘全軍 + 波斯前線接戰者 | InstancedMesh 完整 hoplite | ~7,000 + 數百 |
| 人海層 | 波斯後方主力 | 低面數 InstancedMesh | 數千 |
| 剪影層 | 波斯遠景無邊大軍 | billboard/sprite 群或極簡 box | 視效果填充 |

> **兵力儀表顯示真實史實數字**，遠方群體僅渲染方式降級，數字不縮水。峽谷只容少數人並肩接戰的史實，讓「前線精細、後方人海」在視覺與史實上都自洽。

## 5. 近戰戰鬥呈現系統（本作核心，新模組 `src/engine/melee.js`）

前三場以遠程為主（`effects.js` 全是槍火/曳光/爆炸/彈道），溫泉關是**密集肉搏**，需全新近戰呈現。核心是把所有近戰視覺集中到兩陣相撞的**接戰帶（engagement band）**，後排僅待命微晃 → 動畫成本可控，亦符合「窄道只容少數人並肩」史實。

### 5.1 接戰帶資料模型

由 `engagements.js`／`engage.js` 的每組交戰生成一條 band 物件：

```
band = {
  a, b,            // 交戰雙方單位 ref（希臘 / 波斯）
  center: Vec3,    // 接觸線中心（世界座標）
  dir: Vec3,       // 接觸線法向（指向波斯）
  frontage: 數值,   // 並肩寬度，受地形隘道夾死（中門 ≈ 200 人寬）
  tau: -1..1,      // 拉鋸值（>0 希臘推進、<0 被壓回），吃 engage.js 狀態
  heave: 數值,      // othismos 低頻推擠相位
  intensity: 0..1, // 接戰激烈度 → 驅動 FX 發射率與微觀對決數
}
```

`band.center` 每幀依 `tau` + `heave`（低頻 sin）位移 → 呈現推擠拉鋸的物理感；希臘守 2.5 天 `tau≈+`，第三日轉 `tau→-` 崩潰。

### 5.2 前排動畫（per-instance，`hoplite.js`）

僅接觸線前 `config.meleeDepth`（預設 2）排的士兵啟用：
- **長矛刺擊循環**：矛局部 Z 位移 `= base + A·max(0, sin(t·ω + φ_i))`，`φ_i = hash(i)`（`sin(index)` 雜湊）→ 整道矛林**錯相**起伏、不機械同步。
- **盾牆架擋**：前排盾抬高 + 軀幹前傾出力（`rotX` 微量）。
- **後排待命**：極小幅 idle 擺動，營造「活的方陣」而非靜止方塊。

### 5.3 微觀對決（micro-duels）

沿接觸帶按鄰近度**配對敵我前排士兵**，賦予短狀態機：`lunge（突刺）→ clash（盾撞/格擋）→ resolve（勝者留、敗者倒地）`，單場 ~1–2s、彼此錯開啟動。
- 並發數上限（僅算鏡頭視錐內，預設 ≤ 約 40 對）→ 成本封頂。
- 勝負傾向由 `tau` 加權（希臘優勢時波斯倒地較多）→ 微觀與宏觀拉鋸一致。

### 5.4 倒地與屍牆（溫泉關標誌意象）

- 陣亡 tick 時，接觸線一名士兵轉入 `falling`：~0.6s 內 `rotX→90°` 傾倒，落定為**靜態屍體 mesh**（不再動畫、極省）。
- 屍體**累積成屍堆**橫亙接戰帶前方，堆長/高綁 `sideCasualties` 累積值；波斯兵後續**踩屍前進**（呼應史實）。
- 屍體數設上限 `config.corpseCap`；逾上限時最舊者下沉/淡出，**並 `log` 揭露被裁掉的數量**（沿用專案「不靜默截斷」原則），sources.js caveats 註明為呈現取捨。

### 5.5 近戰特效（`effects.js` 新增 kind='melee'）

取代槍火，沿 band 依 `intensity` 發射：
- `clashSpark`：青銅撞擊火花（短橙白 sprite）
- `dustKick`：接戰帶揚塵
- `bloodPuff`：命中暗紅噴點（**風格化克制**，保留節目感，可由 `config.goreLevel` 調）
- `weaponGlint`：偶發金屬反光
`updateEffects` 依 kind 分派；發射點取 band 接觸線上的取樣點。

### 5.6 箭雨（唯一遠程元素，重用 Normandy 彈道弧線）

波斯弓兵齊射拋物線箭群（細長 instanced 圓柱）；**結局科洛諾斯箭雨**密度拉滿、「遮天蔽日」呼應事件14 對白，守軍在箭雨下逐一倒下歸零（事件46）。

### 5.7 othismos 推擠

整個前段方陣集體湧動：接觸線 + 前排隨 `tau` 前後平移、疊加 `heave` 低頻「一推一收」節奏 → 推擠戰的體感。

### 5.8 專屬 set-piece 編排（storyboard 觸發特化動畫）

| 事件 | 編排 |
|---|---|
| **假退誘敵**(17) | 希臘前排**整齊倒退**→ 波斯追入空檔 →**瞬間回身反殺** + 對決爆發 |
| **推落海**(39) | band 側翼把波斯 instance 朝古海岸推擠**落海**，觸發 `waterSplash`（重用） |
| **矛斷改劍**(40) | 前排武器 mesh `dory→xiphos`，刺擊 `ω` 提高 → 節奏變快變亂 |

### 5.9 效能與鏡頭

- 只動「接觸帶前 `meleeDepth` 排 + 視錐內微觀對決」，後排 idle、屍體凍結 → 動畫量封頂。
- 鏡頭：**越肩**（盾牆後望出）、**貼地低角**、列奧尼達戰死(41)**慢動作**；由 storyboard 運鏡。

### 5.10 模組接線

新模組 `src/engine/melee.js`：`S.initMelee()/updateMelee(t)`——由 engagements 建 band、每幀更新 `tau/heave/intensity`、排程微觀對決與倒地堆屍、經 `effects.js` 發射近戰 FX。讀 `engage.js` 拉鋸與 `sideCasualties`。`thermopylae.html` 掛 script、`thermopylae-main.js` init/update 接線。向後相容，前三場不觸發。

## 6. 電影化運鏡與子彈時間（Bullet-time + 動態運鏡）

近戰的張力來自「慢下來看清楚」+「鏡頭繞著動作走」。本作把它做成一套獨立的電影化層，疊在既有 storyboard 運鏡之上。

### 6.1 時間縮放層（cinematic time-scale）

- 全域 `cinemaScale ∈ (0,1]` 乘上餵給**動畫系統的 dt**（melee/hoplite 動畫/effects/箭矢飛行）。`1`=正常、`~0.08`=子彈時間。
- **與使用者播放倍速解耦**：倍速縮放的是「歷史時間軸 t」（三日進度），`cinemaScale` 只是電影化覆蓋層，兩者獨立相乘。
- 進出皆**緩動**（ease-in 變慢 → hold → ease-out 回復），絕不瞬間切換；變慢時歷史時間軸 t 同步進入既有 **hold**（沿用 Normandy 的 hold 平滑機制），形成「凝住的一瞬」。

### 6.2 子彈時間節拍（綁關鍵事件，storyboard 觸發）

| 事件 | 慢動作編排 |
|---|---|
| **17 回身反殺** | 假退到位的瞬間 `cinemaScale→0.12`，鏡頭甩到斯巴達回身的矛尖 |
| **40 矛斷改劍** | 長矛折斷一刻定格慢放，火花飛濺 |
| **41 列奧尼達戰死** | `cinemaScale→0.08`、real ~2.5s，**環繞慢動作**＋推近 → 全片情緒最高點 |
| **42 奪回遺體** | 四度拉鋸的最後一奪，慢放定格 |
| **46 科洛諾斯箭雨** | 箭雨**落下過程**整段慢放，箭矢逐支插落、守軍逐一傾倒 |

### 6.3 動態運鏡（鏡頭隨動作轉動 + 縮放）

`storyboard.js` 擴充關鍵影格模式（皆可緩動插值）：
- `track`：鏡頭跟著 band 中心／焦點平移
- **`orbit`**：繞焦點公轉（角度 θ(t) 掃動、半徑 r）——**子彈時間的招牌環繞鏡頭**
- **`push`/`pull`**：dolly 推近／拉遠（改 camera 距離）
- **`fov`/dolly-zoom**：必要時做 vertigo 變焦衝擊
- 焦點來源：`melee.js` 對外暴露焦點（列奧尼達座標、band 中心、奪屍標點、科洛諾斯小丘）

子彈時間發生時 = **`orbit` 環繞 + `push` 推近**同時進行，時間幾近凝止、鏡頭繞著定格的刺殺/倒地公轉一圈再拉回。

### 6.4 平時運鏡也保持「活」的

即使非慢動作，鏡頭也持續轉動縮放跟戰況：越肩 → 緩慢 `orbit` → `push` 貼近接戰帶；鏡頭從不靜止呆滯。沿用既有平滑 camera target + hold，避免生硬跳切。

### 6.5 接線

- `cinemaScale` 置於 `thermopylae-main.js` 主迴圈：`updateMelee(dt*cinemaScale)`、`updateEffects(dt*cinemaScale)`、hoplite 動畫同享；storyboard 控制 `cinemaScale` 曲線 + 鏡頭影格。
- 全部走既有 config／storyboard 機制，前三場不設這些節拍 → 不受影響。
